# Samadhaan — TestNG Reports Demo

This folder is a **standalone Java + Maven + TestNG** project created solely to
demonstrate TestNG report generation for the Samadhaan Complaint Management System.

> **It does NOT modify the React frontend, Node.js backend, MongoDB, or the existing
> Mocha/Selenium tests.** It runs in parallel as a separate test artefact.

---

## 📁 Folder Structure

```
testng-reports-demo/
├── pom.xml                  ← Maven build file (Selenium + TestNG deps)
├── testng.xml               ← TestNG suite definition
├── build.xml                ← ANT build file for XSLT report
├── testng-results.xsl       ← Custom XSLT stylesheet
├── src/
│   └── test/
│       └── java/
│           └── SamadhaanTest.java   ← TestNG test class (5 test cases)
└── test-output/             ← Auto-generated after mvn test
    ├── emailable-report.html
    ├── index.html
    └── XSLT_Report.html     ← Generated by ant report
```

---

## ✅ Prerequisites

| Tool | Version | Install |
|------|---------|---------|
| Java JDK | 11+ | `brew install openjdk@11` |
| Maven | 3.6+ | `brew install maven` |
| ANT | 1.10+ | `brew install ant` |
| Google Chrome | latest | Download from google.com |

> WebDriverManager auto-downloads the matching ChromeDriver — no manual setup needed.

---

## 🚀 Step 1 — Start the Samadhaan Frontend

The tests navigate to `http://localhost:5173`, so the frontend must be running.

```bash
cd ../frontend-new
npm run dev
```

---

## 🧪 Step 2 — Run TestNG Tests (Maven)

```bash
cd testng-reports-demo
mvn test
```

This compiles `SamadhaanTest.java`, runs all 5 test cases, and generates the
following reports **automatically** inside `test-output/`:

---

## 📊 Report Locations

### 1. Emailable Report
```
test-output/emailable-report.html
```
A compact, self-contained HTML file. Open in any browser or email it directly
as an attachment for lab/viva demonstrations.

### 2. Index Report (Full)
```
test-output/index.html
```
The full TestNG HTML report with expandable suites, test logs, and detailed
method-level results. Best for detailed viewing.

### 3. XSLT Report (via ANT)

After `mvn test`, run:

```bash
ant report
```

Output:
```
test-output/XSLT_Report.html
```

A **custom, dark-themed styled report** showing:
- Total Tests
- Passed count
- Failed count
- Skipped count
- Individual test results with duration
- Failure exception messages

## 🧩 Test Cases Summary

| # | Method | Expected | Purpose |
|---|--------|----------|---------|
| TC-01 | `testHomepageLoads` | ✅ PASS | Verifies the Samadhaan homepage loads and contains "samadhaan" in source |
| TC-02 | `testLoginPageRenders` | ✅ PASS | Verifies the login form renders with email input |
| TC-03 | `testDashboardRedirectsToLogin` | ✅ PASS | Unauthenticated `/dashboard` → `/login` |
| TC-04 | `testProfileRedirectsToLogin` | ✅ PASS | Unauthenticated `/profile` → `/login` |
| TC-05 | `testIntentionalFailForDemoReport` | ❌ FAIL (intentional) | Demonstrates FAIL state in reports |

> To disable the intentional failure, set `enabled = false` on `@Test` in TC-05.

---

## 🔧 ANT Command Reference

```bash
# Generate XSLT report (requires test-output/testng-results.xml to exist)
ant report

# Clean generated XSLT report
ant clean
```

---

## 📝 Notes

- All reports are in `test-output/` which is created automatically by TestNG.
- The `testng-results.xml` file (input for the XSLT report) is also auto-generated.
- Chrome runs in **headless mode** by default. To see the browser, remove the
  `--headless=new` argument from `SamadhaanTest.java → setUp()`.
